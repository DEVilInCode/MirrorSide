# 3 Системные требования
## 3\.1 Функциональные требования
### 3\.1.1 Вход в игру

Описание. Пользователь имеет возможность зарегистрироваться с помощью почты, либо войти в существующий аккаунт.

|Функция|Описание|
| :- | :- |
|Регистрация пользователя|Игра должна запросить пользователя ввести все необходимые параметры(имя, почта и пароль). Пользователь может выйти с окна регистрации, либо заполнить поля и создать аккаунт|
|Пользователь с таким именем существует|Игра должна известить пользователя о том, что имя уже занято и запросить ввод снова. Пользователь может выйти, либо продолжить заполнять поля.|
|Количество символов пароля слишком маленькое.|Игра должна известить пользователя о том, что пароль должен состоять из 6 и более символов. Пользователь может выйти, либо изменить пароль под условия.|
|Аккаунт на данную почту уже существует|Игра должна известить пользователя, что на данную почту уже завели аккаунт и ему нужно перейти в окно входа. Пользователь должен закрыть окно регистрации и перейти в окно входа|
|Подтверждение пароля|Пользователь должен продублировать придуманный пароль в специальное поле. Игра должна сравнить пароли и известить пользователя о том, верно ли заполнены поля паролей. В случае несовпадения, обнулить второе поле. Пользователь может снова заполнить его, либо выйти.|
|Вход зарегистрированного пользователя|Игра должна предоставить пользователю окно входа. Она должна запросить ввод почты и пароля. При верном заполнении перейти к запуску главного экрана, иначе известить пользователя о неверно заполненных полях. Пользователь может повторить попытку входа заново заполнив поля, либо выйти. |

### 3\.1.2 Возможности главного экрана


|Функция|Описание|
| :- | :- |
|Главное меню|Пользователь имеет возможность открыть главное меню. Игра должна предоставить пользователю кнопку позволяющую перейти в окно главного меню. Главное меню должно содержать вкладки: магазин, коллекция и сборка колод.|
|Магазин|Пользователь имеет возможность приобрести новые карты, усиления и прочее. Игра должна предоставить список товаров за внутри игровые ресурсы, отображать иконки и цену. Пользователь может посмотреть информацию о товаре и приобрести его, либо нет.|
|Коллекция|Пользователь может просмотреть список и количество собранных им карт. Игра должна предоставить список предметов имеющихся у пользователя, а также поле для поиска определённого предмета. Также игра должна отображать категории предметов для упрощённого поиска. |
|Сборка колод|Пользователь имеет возможность собрать колоду с помощью которой он может проходить компанию или сражаться в PvP. Игра должна отображать уже имеющиеся колоды пользователя, а также кнопку для добавления новых.|
|Настройки|Пользователь имеет возможность открыть и изменить настройки графики и уровня звука. Игра должна предоставить пользователю иконку настроек, а также отобразить окно с текущими настройками.|
|Профиль пользователя|Пользователь может посмотреть свой текущий профиль изменить аватар аккаунта, посмотреть статистику и возможность изменить имя пользователя. Игра должна предоставить окно, содержащее поля (аватар, статистика, имя пользователя).|
|Сражения|Пользователь может выбрать режим сражения: компания, PvP, подземелья. Игра должна предоставить всплывающие иконки режимов и возможность выбрать любой.|

## 3\.2 Нефункциональные требования
### 3\.2.1 Атрибуты качества
### <a name="user-content-requirements_for_ease_of_us"></a>3.2.1.1 Требования к удобству использования
1. Приложение реализует все основные функции;
1. Все функциональные элементы пользовательского интерфейса имеют названия, описывающие действие, которое произойдет при выборе элемента.
1. Интерфейс не содержит лишних элементов.
### <a name="user-content-security_requirements"></a>3.2.1.2 Требования к безопасности
1. Приложение предоставляет доступ только зарегистрированным пользователям.
### <a name="user-content-access_requirements"></a>3.2.1.3 Требования к доступности
1. Время реакции на действия пользователя минимально.
### <a name="user-content-external_interfaces"></a>3.2.2 Внешние интерфейсы
1. Окна приложения удобны для использования пользователями:
1. Размер шрифта не должен быть слишком маленьким;
### <a name="user-content-restrictions"></a>3.2.3 Ограничения
1. Приложение реализовано для ОС Android;
1. Язык, на котором реализована программа, - С#/GDScript.
